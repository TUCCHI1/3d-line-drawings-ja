構造化データで検索エンジンと会話する技術

📚 application/ld+json とは？

JSON-LD (JSON for Linking Data) は、Webページの内容を機械可読な形式で記述する標準フォーマットです。

🎯 なぜ必要？

HTML は人間が読むため、検索エンジンは「これは商品名なのか？価格なのか？」を完全には理解できません。JSON-LD を使うと、明示的に情報を伝えられます。

🏗️ 基本構造

HTMLの<script>タグ内に記述します：

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Product", "name": "商品名" } </script>

重要なキー：

  • @context - 語彙の定義元（ほぼ常に "https://schema.org"）
  • @type - データの種類（Product, Article, Personなど）
  • その他のフィールドは@typeによって変わる

🛠️ 実装方法

1. HTMLに直接記述

<!DOCTYPE html> <html> <head> <script type="application/ld+json"> { "@context": "https://schema.org", "@type": "WebPage", "name": "ページタイトル" } </script> </head>

2. Nuxt 3 で useHead を使う

useHead({ script: [ { type: 'application/ld+json', innerHTML: JSON.stringify({ '@context': 'https://schema.org', '@type': 'Product', 'name': '商品名' }) } ] })

3. Next.js で実装

export default function Page() { const jsonLd = { '@context': 'https://schema.org', '@type': 'Product', 'name': '商品名' }; return ( <> <script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }} /> </> ); }

✨ ベストプラクティス

✅ DO（推奨）

  • 必須フィールドは必ず含める
  • 正確な情報を記述する（虚偽情報はペナルティ対象）
  • 画像は高解像度（最低 1200px推奨）
  • 日付はISO 8601形式（2024-10-30）
  • 複数のタイプを使う場合は配列で
  • 定期的にテストツールで検証

❌ DON'T（非推奨）

  • ページ内容と異なる情報を記述
  • 過度に多くのタイプを設定
  • レビューの捏造
  • スパム的なキーワードの羅列
  • 未公開コンテンツの情報を含める
